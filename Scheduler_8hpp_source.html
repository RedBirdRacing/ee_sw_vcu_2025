21/**
22 * @brief Scheduler class template for scheduling tasks on multiple MCP2515 instances
23 * Takes in function pointers to member functions of MCP2515 class,
24 * and calls them at specified intervals, using a unified central ticker.
25 *
26 * @details The Scheduler class holds function pointers to tasks to be run on multiple MCP2515 instances.
27 * Because how most MCP2515 are dedicated to a specific function (e.g., motor control, BMS, datalogger),
28 * instead of the scheduler holding a queue of messages to send and a queue of read messages,
29 * it holds function pointers to tasks that handle sending and receiving messages for each MCP2515.
30 * The object wishing to send/receive messages holds their own MCP2515 object (reference).
31 *
32 * The Scheduler runs on a fixed period. When Scheduler.update() is called, it checks if a period has already passed.
33 * If not, it checks if it is almost passed. If the time to the next cycle is less than SPIN_US, it spin-waits until the period is reached.
34 * Otherwise, it returns immediately, allowing other non-scheduler tasks to run.
35 * Once the time to fire is reached, it runs the tasks via the pointers in a round-robin fashion, going from one MCP2515 to another.
36 *
37 * With a modified mcp2515.h that doesn't directly use SPI.h, we can skip waiting for the SPI transaction to complete,
38 * and instead work on compiling the next message while the previous SPI transaction is still ongoing.
39 * Although it would be easier to only have a single queue (instead NUM_MCP2515 queues),
40 * we give room for the CAN-bus to be busy on one MCP2515, while another MCP2515 can still send/receive messages,
41 * i.e. we distribute the load across multiple CAN buses more evenly, instead of having one bus burst at one time
42 *
43 * In the rare case where the system is busy and misses more than one period, the scheduler will skip to the next period, preventing bursts.
44 *
45 * @tparam NUM_TASKS Number of tasks per MCP2515, choose highest of all, but keep as low as possible
46 * @tparam NUM_MCP2515 Number of MCP2515 instances
47 */
